# OTP AEO Question Map

**Purpose:** the source of truth for OTP's Answer-Engine-Optimization editorial calendar. Every post in the Organizing Agents series traces back to one of the questions on this map.

**Strategy:** dominate AI engine citations for "how do I organize / measure / coordinate / staff / pay / govern / scale AI agents inside a company" before any other source claims the topic. Each post answers one specific question with concrete examples from the live Sneeze It hybrid chart, schema-marked-up, and ends with the OTP MCP install snippet so visitors can interact with the same chart from their AI assistant within 30 seconds of finishing the article.

**Cadence:** 3 posts per day, weekdays. Daily ritual via the `/OTP-AEO` skill.

**Voice gate:** every post in David Steel's voice — declarative, no em dashes, short paragraphs, concrete examples drawn from the Sneeze It chart, contrarian or clarifying central claim, ends with the install snippet.

## Daily prioritization rule

Each day pick from the cluster currently underrepresented relative to its priority weight. Cluster 1 and Cluster 2 should always be at the front of the line. Cluster 11 and 12 should be sprinkled in once per week to broaden the topical authority signal beyond pure how-to.

## Cluster interlink strategy

Every published post should link to two other published posts in the same cluster (when available) and one in an adjacent cluster. The internal link graph is what tells AI engines "this domain has real depth on this topic."

## Schema strategy

Every post gets `BlogPosting` schema (auto). Every post with a numbered list of items gets `HowTo`. Every post that opens with a literal question gets `FAQPage` markup added in the article body. The `/blog/:slug` route already wires the BlogPosting schema; HowTo and FAQ enhancements are post-by-post additions in the markdown content.

## Status tracking discipline

Each row in this map gets updated with a status flip and live URL the moment the post ships. The `/OTP-AEO` skill enforces this. Stale "drafted" rows are red flags.
